# Глоссарий

## Телеком-домен
- **HLR (Home Location Register)**: база данных абонентов мобильной сети. Требует быстрых чтений и обновлений.
- **CDR (Call Detail Record)**: запись о событии (звонок, SMS). Пишется огромным потоком, читается редко.
- **IMSI**: уникальный идентификатор SIM-карты (International Mobile Subscriber Identity). Обычно используется как ключ в БД.
- **Fraud Detection**: система выявления мошенничества. Требует быстрой проверки каждого события на наличие в "черных списках".

## Базы данных и Алгоритмы
- **KV (Key-Value)**: хранилище пар ключ–значение.
- **Memtable**: буфер в памяти, где накапливаются новые данные перед записью на диск (обычно SkipList или Tree).
- **SSTable (Sorted String Table)**: неизменяемый файл на диске, где данные лежат отсортированно.
- **WAL (Write Ahead Log)**: журнал на диске, куда пишутся данные *до* попадания в память, чтобы пережить сбой питания.
- **LSM Tree (Log-Structured Merge Tree)**: архитектура БД, оптимизированная под запись (Sequential Write). Состоит из Memtable, WAL и SSTables.
- **Compaction**: фоновый процесс слияния нескольких мелких SSTable в один большой (уборка мусора).
- **Bloom Filter**: вероятностная структура, позволяющая быстро сказать "ключа точно нет", не читая диск.
- **Write Amplification**: во сколько раз реальный объем записи на диск превышает объем полезных данных (из-за WAL и Compaction).
- **Consistent Hashing**: алгоритм распределения данных по узлам кластера, минимизирующий пересылку данных при изменении числа узлов.
